# v1.0.3: expand non-Pi update coverage, harden CI quality gates, and add agent customization pack

### Summary

This release improves update behavior on non-Raspberry Pi Linux systems, modernizes CI dependency and lint policy enforcement, and introduces complete AI agent/skill customization files for contributor workflows.

### What Changed

#### 1. Installer and Runtime Behavior

- Enabled non-Pi flows to keep firmware update capability while excluding Pi-only update paths where appropriate.
- Improved installer input handling and non-interactive safety behavior.
- Preserved and strengthened cron-safe self-update execution flow.

Files:

- `install.sh`
- `scripts/update_pi_firmware.sh`
- `scripts/update_pip.sh`
- `scripts/update_pi_os.sh`
- `scripts/update_pi_apps.sh`
- `scripts/docker_cleanup.sh`
- `scripts/update_self.sh`
- `scripts/update_samsung_ssd.sh`
- `uninstall.sh`

#### 2. Test Coverage and Reliability

- Added dedicated non-Pi installer mode tests.
- Expanded and hardened mocks used by BATS test suites.
- Refined test runner behavior and coverage collection paths.

Files:

- `tests/install_non_pi_mode.bats` (new)
- `tests/component_tests.bats`
- `tests/run_suite.sh`
- `tests/setup_mocks.sh`

#### 3. Linting, Formatting, and Policy Enforcement

- Added strict, multi-tool lint orchestration with mandatory markdown formatting checks.
- Enforced 140-char max length for shell, YAML, and Dockerfiles.
- Explicitly preserved no markdown max-line-length rule while keeping markdown lint mandatory.
- Added formatter entrypoint for shell scripts.

Files:

- `tests/lint.sh`
- `tests/format.sh` (new)
- `.shellcheckrc` (new)
- `.yamllint.yml` (new)
- `.hadolint.yaml` (new)
- `.pymarkdown.json` (new)
- `.editorconfig` (new)

#### 4. CI and Dependency Hygiene

- Upgraded CI action dependencies to current stable versions.
- Improved lint job setup and strict lint execution behavior.
- Added Dependabot updates for GitHub Actions and Docker.
- Refined Docker test image build details for reliable execution.

Files:

- `.github/dependabot.yml` (new)
- `docker/Dockerfile.test`
